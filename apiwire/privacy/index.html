Privacy

Local by default. Sync only when you choose it.

API Wire is designed as a desktop app with local data handling by default. Core local usage does not require a cloud account, and this repository does not publish a default AANKDA-operated sync endpoint.

Effective date: March 8, 2026

What stays on your device

By default, API Wire processes and stores its working data on your device, including request collections and history, local environment values and settings, and local diagnostics and log data.

What is not sent by default

When you use API Wire in local-only mode, it does not send your collections, workspace environment metadata, or local settings snapshots to a remote service by default.

Cloud Sync is optional

If you configure a sync endpoint and choose to use Cloud Sync, API Wire can transmit collection data included in a workspace snapshot, workspace environment metadata, and selected local settings snapshots.

Secrets and secure values

Secrets are intended to be stored through local secret-handling paths such as OS keyring integrations. Secure values are not exported from workspace metadata as plaintext.

No default AANKDA backend

This repository does not publish a default AANKDA-operated sync endpoint. If you point Cloud Sync to an endpoint run by your organization or a third party, that operator's privacy terms apply separately.

Your control

You control the local data files stored by API Wire and can remove them from your device at any time. Privacy questions can be sent to privacy@aankda.ai.

Privacy behavior changes? This page changes with it.

If privacy behavior changes materially, this page is updated to reflect it. Questions go to privacy@aankda.ai.
